Architect Business Model v1.0

Free tool + token consumption revenue model

Key Takeaways

  1. Free tool, paid tokens. Architect is free to download and use. Revenue comes from AI token consumption at a 6x markup on inference cost, creating a usage-based model similar to AWS, Twilio, and Stripe.
  2. 83% to 91% gross margins. COGS drops from 17% (solo tier) to 9% (enterprise tier). Blended margins improve as the customer mix shifts upmarket.
  3. 80% retention rate. Conservative versus the 90% SaaS industry average. Even at 80%, the installed base compounds rapidly because new cohorts layer on top of retained ones.
  4. First revenue: Q2 2026 (April). Product launches with paying customers in April 2026 after a closed beta period.
  5. Eight distinct revenue streams beyond token usage: fine-tuning, agent packs, custom pipelines, on-prem deployment, starter packs, data-ownership licensing, SLA, and onboarding.

How Architect Makes Money

Architect follows a free distribution, usage-based monetization model. The base tool costs nothing. Any studio can download it and start converting lore into structured game data immediately.

Revenue activates when users run AI-powered features: generation, suggestion, extraction, branching, and dialogue construction. Each of these operations consumes tokens. LoreWeaver applies a 6x markup on the underlying AI inference cost.

This model mirrors proven infrastructure businesses:

Company What is free What is paid
AWS Account creation, free tier Compute, storage, bandwidth by usage
Twilio Developer account, sandbox API calls (SMS, voice, video) by usage
Stripe Integration, dashboard Transaction processing by volume
LoreWeaver Architect Full authoring tool AI token consumption by usage

Why free distribution is the right strategy for a dev tool:

  1. Game studios evaluate tools by trying them. A free tool eliminates procurement friction and lets narrative designers adopt Architect without budget approval.
  2. Once a studio builds workflows around Architect, switching costs are high. The structured data formats, quest pipelines, and dialogue trees become embedded in the production process.
  3. Token consumption scales naturally with project size. A studio working on a 40-hour RPG consumes far more tokens than a studio prototyping a 2-hour narrative experience. Revenue scales with the value delivered.
  4. Free distribution creates a large installed base that generates word of mouth, community content, and marketplace activity, all of which reduce customer acquisition cost over time.

Why margins improve at scale: Larger customers consume more tokens but require proportionally less support, onboarding, and infrastructure overhead per euro of revenue. The 6x token markup provides a thick margin cushion, and the fixed cost components (platform hosting, core engineering) do not scale linearly with customer count.

Target blended gross margin at maturity: ~91%.

Growth Flywheel

Architect's business model creates a self-reinforcing cycle:

Stage 1: Free tool drives adoption. Zero cost removes every barrier to trial. Narrative designers download Architect, try it on a side project, and bring it into their studio's workflow.

Stage 2: Adoption drives token consumption. As studios integrate Architect into production, they use AI features more heavily. A studio that starts by generating a few dialogue trees will eventually run full quest pipelines, character arc extraction, and branching narrative generation.

Stage 3: Token consumption drives revenue. The 6x markup on inference cost converts usage into high-margin revenue. Revenue scales with the complexity and size of each studio's projects.

Stage 4: Revenue funds product development. More features (new AI agents, better generation quality, additional engine integrations) make the tool more valuable, which attracts more studios and increases usage per existing studio.

The flywheel accelerates because each new studio that adopts Architect also contributes to the community: sharing templates, building marketplace plugins, and creating tutorials that lower the barrier for the next studio.
