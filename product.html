1. Architect, live today

Architect compresses narrative production workflow into structured, engine-ready output. The product is in working beta with 30 users on it now.

Workflow: Input to AI Processing to Output

Stage What happens Output
Input Upload raw lore documents, design notes, world bible, free-form prose. Project workspace seeded with unstructured source material.
AI Processing 15 structured AI operations across entity extraction, relationship inference, schema generation, narrative board structuring, conflict modeling. Validated entities, relationships, quests, scenes ready for review.
Output Export to engine-ready formats (JSON, XML, CSV) with schema validation built in. Game-ready narrative data, drop-in for Unity, Unreal, Godot pipelines.

The cost compression that matters

Manual (narrative designer plus tech writer) €41,250 6 to 8 weeks

The savings are not from removing the writer. They are from removing the structured production work: tagging, schema fitting, validation, export, format conversion, change tracking. The writer retains creative authorship; Architect removes the engineering tax around it.

2. Director, on-device runtime

Director is the runtime layer. It executes Architect-authored narrative inside the game engine, on-device, with no cloud inference cost and no telemetry leaving the player's machine.

The runtime takes authored narrative state (entities, relationships, mutable/immutable parameters) and lets the LLM make decisions inside a validated schema. The engine sees only schema-valid responses. The LLM never gets to break game state.

This solves the structural problem cloud NPC engines cannot solve:

  • No cloud inference cost. Cloud competitors charge per interaction. At AAA scale this is prohibitive. On-device is zero marginal cost.
  • 1.5s TTFS at consumer hardware targets. Cloud round-trips alone are 1000ms or more before any generation.
  • No player data leaves the device. Studios cannot ship cloud NPC chat on console without manufacturer approval that does not exist.
  • Schema-validated output. Game designers control exactly what the AI can and cannot change.

Business model: free SDK, 1% royalty on net revenue above €100K per title. Architect users get Director integration as part of the pipeline.

3. Defensibility

Four overlapping defensibility layers, ranked by how hard each is to copy.

The schema contract

The conflict resolution XML schema with mutable/immutable parameter enforcement is the single most defensible technical asset. It is the only formally specified, validated contract for game-state-to-LLM interaction that exists in the market.

Capability What it does
XSD-validated contract Formal schema defining how game engines communicate with the LLM.
Mutable / immutable flags Designers declare exactly what the AI can and cannot change at runtime.
Response validation Type coercion, range checking, valid-value enforcement before results reach game logic.
Round-trip format matching JSON or XML in, same format out.

Inworld and Convai use free-form dialogue. NVIDIA ACE uses proprietary internal formats. None of them give designers schema-level control over AI behavior. If studios adopt the LoreWeaver schema, it becomes a de facto standard with switching costs measured in re-authoring entire narrative packs.

Creator plus player telemetry flywheel

Architect captures authored structure: entities, relationships, intents, schema choices. Director captures runtime telemetry: what schemas hold up under play, which mutable parameters get exercised, which decisions break immersion. Both feed back into the model and into the next iteration of the product.

Competitors have one side or the other. LoreWeaver has both, and the two compound. Better authoring leads to better runtime decisions, which build trust, which drives deeper adoption of the authoring layer.

Pipeline integration switching costs

Once a studio builds its world in Architect, the narrative data is shaped around Architect's schemas. Switching means re-authoring. Director multiplies that lock-in: a shipped title using Director's runtime contract cannot trivially swap to a competitor without rebuilding all runtime integration. Standards create gravity.

Cross-domain expertise

The team combines AI, game development, and narrative design. Most competitors approach from one angle: AI teams that understand models but not game production, tooling teams that understand pipelines but not emergence, or narrative teams that understand story but not systems. The combination is rare and explains why everyone else builds NPC chat: it is the obvious product when you understand only LLMs.

4. The combined narrative stack

Architect and Director are designed as one system. Architect's structured output is exactly the input shape Director needs to run constrained on-device emergence well. Owning both sides compounds in three ways.

  1. Data quality. Director does not need a larger model to outperform competitors. It needs better upstream data. Architect provides it.
  2. Schema standardisation. Architect emits the contract; Director enforces it. The same schema definitions live across authoring and runtime. Competitors cannot ship this without rebuilding both sides.
  3. Workflow lock-in. Studios building in Architect ship through Director because the integration cost is zero. Studios shipping through Director author in Architect because nothing else emits the right schema.

The combined stack is the moat. Single-product competitors face the choice of building authoring (years of game-tooling work) or runtime (years of game-engine work) before they can compete on the integrated story. LoreWeaver has both in market.
